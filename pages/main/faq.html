3. Automatisation & Tests

5 questions

  • Structure minimale :

    *** Settings ***
Library    SeleniumLibrary

*** Test Cases ***
Vérifier la page d'accueil
    Open Browser    ${LOGIN_URL}    chrome
    Title Should Be    Portail de Test
    [Teardown]    Close Browser

    Enregistre le fichier en .robot puis exécute robot ton_test.robot.

    1. Installe les dépendances : pip install robotframework seleniumlibrary.
    2. Installe le driver (ChromeDriver ou GeckoDriver) compatible avec ton navigateur.
    3. Dans un terminal, exécute robot chemin/vers/test.robot.

    Consulte les rapports générés (log.html, report.html) pour analyser les résultats.

    • report.html : synthèse (succès, échecs, durée).
    • log.html : détails étape par étape (keywords exécutés, captures).
    • Utilise la recherche pour filtrer une suite ou un mot-clé spécifique.

    Archive les rapports après chaque sprint pour conserver l'historique.

    1. Installe l'extension Selenium IDE sur Chrome ou Firefox.
    2. Crée un nouveau projet, clique sur Record et navigue sur ton application.
    3. Stoppe l'enregistrement, puis édite/commente les étapes avant export.

    Pense à nettoyer les sélecteurs pour les rendre robustes.

    • Préférez les attributs stables (data-testid, aria-label).
    • Évitez les indices (div[3]) et les chemins absolus.
    • Centralisez vos sélecteurs dans des mots-clés réutilisables.

    Documente chaque sélecteur dans le repo pour faciliter la maintenance.
